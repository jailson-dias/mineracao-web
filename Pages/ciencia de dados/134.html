Questões Comentadas – Informática para Polícia Federal – Agente – Banco de dados

Comentário: Em seu livro Teoria geral dos sistemas, esse autor apresenta a teoria e tece considerações a respeito de suas potencialidades na física, na biologia e nas ciências sociais. No mesmo livro, von Bertalanffy lança os pressupostos e orientações básicos de sua teoria geral dos sistemas, como segue:

  1. a) há uma tendência para a integração nas várias ciências naturais e sociais;
  2. b) tal integração parece orientar-se para uma teoria dos sistemas;
  3. c) essa teoria pode ser um meio importante de objetivar os campos não-físicos do conhecimento científico, especialmente nas ciências sociais;
  4. d) desenvolvendo princípios unificadores que atravessam verticalmente os universos particulares das diversas ciências, essa teoria aproxima-nos do objetivo da unidade da ciência;
  5. e) isso pode levar a uma integração muito necessária na educação científica.

Logo, temos nossa resposta na alternativa C.

Gabarito extraoficial: 73. C

Comentário: Vejamos a justificativa para cada uma das afirmações acima.

  1. Vejam que essa alternativa coloca o conhecimento como uma etapa posterior a informação dentro de um fluxo de encadeamento e complexidade. O conhecimento, conforme falado na nossa revisão, trata de informações úteis a um propósito, envolve entendimento e análise de um conjunto de informações. Sendo assim, podemos marcar a assertiva como correta.
  2. Essa assertiva fala do fluxo de desenvolvimento de um sistema de informação. Sabemos que a etapa de engenharia de requisitos visa o alinhamento das demandas de projeto por parte dos clientes com a equipe que vai desenvolver o sistema. O resultado desta interação é um documento de requisitos que descreve os aspectos funcionais e não funcionais do sistema. Na etapa seguinte, a equipe de desenvolvimento fará um detalhamento dos requisitos agregando informações, definido a arquitetura do sistema e o banco de dados. Logo, a alternativa está, de fato, correta.

Gabarito extraoficial: 79. C  80. C

Comentário: Vamos comentar cada uma das afirmações acima.

  1. Temos que lembrar que a leitura da cardinalidade do relacionamento entre duas entidades é feita de forma “cruzada”, neste caso, cada produto está associado a um tipo de produto e cada tipo de produto pode estar associado a vários produtos. Logo, temos uma alternativa incorreta.
  2. Quando vamos passar do modelo ER para o relacional um relacionamento 1:N, precisamos pegar a chave primária do lado 1 e colocar como chave estrangeira do lado N. No exemplo, temos que usar a chave primária (código) associada ao tipo de produto e inserir um novo campo na relação produto que será uma chave estrangeira e aponta para o código do tipo de produto (na tabela tipo de produto). A fusão de tabelas geralmente é usada quando tempos um relacionamento obrigatório 1:1. Sendo assim, temos uma alternativa incorreta.
  3. Para responder a essa questão pense no atributo CPF. Ele pode aparecer em várias tabelas de um mesmo modelo de dados, como atributo ou ainda como chave primária. Ou seja, não existe problema em colocar atributos com mesmo nome em tabelas distintas. Logo a alternativa está incorreta.

Gabarito extraoficial: 81. E 82. E 83. E

Comentário: Vamos comentar cada uma das afirmações acima.

  1. A questão apresenta uma definição consistente com o conceito de mineração de dados. O texto fala da descoberta de padrões úteis sobre dados. Logo, alternativa correta.
  2. A situação descrita apresenta conceitos que nos levam a um modelo de aprendizado supervisionado. Veja que o texto fala na presença de classes, previamente definidas, que são usadas para rotular novos elementos de dados. Na assertiva, ele afirma que a situação nos leva a um processo ou tarefa de classificação. Logo, a alternativa está correta.
  3. Mais uma vez, temos uma definição correta de Big Data. Veja que a afirmação aborda os 3 V’s presentes na definição original: volume, variedade e velocidade. Logo, temos uma alternativa correta.

Gabarito extraoficial: 84. C 85. C 86. C

Comentário: Vamos comentar as alternativas acima.

  1. Neste caso temos 2 vetores com 3 elementos cada um. A operação x + y vai fazer o somatório dos de cada um dos termos. Veja o código abaixo:

> x <- c (3, 5, 7)

> y <- c (1, 9, 11)

> print (x + y)

[1]  4 14 18

Neste caso o valor retornado pela função print vai retornar como resultado o vetor 4 , 14 e 18. Logo, a alternativa está incorreta.

  1. Nesta alternativa atribuímos valores booleanos as variáveis x e y. Em seguida, pedimos para imprimir o atributo xy, que não existe no sistema. Isso retornar um erro conforme visto abaixo.

> x <- TRUE

> y <- FALSE

> print (xy)

Error in print(xy): object ‘xy’ not found

Logo, temos mais uma alternativa errada.

Gabarito extraoficial: 93. E 94. E

Comentário: Vamos comentar as alternativas acima.

  1. O controle de bloco em Python é feito com indentação exclusivamente. Não é permitido o uso de chaves. Falamos sobre isso no vídeo, no PDF e na revisão.
  2. Mesmo motivo da 95, e, além disso, ainda tem um “==” fazendo atribuição. Atribuição em Python é feita com um igual (“=”) e não com dois. O segundo serve para o teste de igualdade. Logo, temos uma alternativa E.

Gabarito extraoficial: 95.E 96.E

Qualquer dúvida estou às ordens,

Thiago Rodrigues Cavalcanti

Analista do Banco Central do Brasil na Área de Tecnologia da Informação (TI). Formado em Ciência da Computação na Universidade Federal de Pernambuco - UFPE. Mestrado em Engenharia de Software pela UPFE. Professor de Tecnologia da Informação para concursos e cursos de TI. Possui grande experiência em concursos públicos, tendo sido aprovado e convocado em diversos concursos de expressão como BNDES, ANAC, TCE-RN, Infraero, Decea-RJ e INMETRO-RS. Trabalhou com modelagem e administração de dados, contrução de DWs e ferramentas de análise de dados voltadas para consolidação de informações sobre o sistema financeiro nacional. Atualmente é aluno de doutorado em economia na UnB.
